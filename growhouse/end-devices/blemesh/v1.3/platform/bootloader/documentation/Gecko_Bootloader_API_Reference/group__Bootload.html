bool bootload_verifyApplication ( uint32_t  startAddress)

Verify the app starting at address startAddress

If secure boot is enforced, the function will only return true if the cryptographic signature of the application is valid. Else, the application is validated according to the signature type present in the application ApplicationProperties_t structure.

Examples without secure boot:

  • App has no signature: Valid if initial stack pointer and program counter have reasonable values
  • App has CRC checksum: Valid if checksum is valid
  • App has ECDSA signature: Valid if ECDSA signature is valid
Parameters
[in]startAddressStarting address of the application
Returns
True if the image is deemed valid
